# LLM Wiki Integration Brief

Use this before editing when the source is large, ambiguous, high-impact, or likely to update more than five pages.

```markdown
## Integration Brief

### Source

- title:
- path or URL:
- author or publisher:
- date:

### Extraction

- Key claims:
- Entities:
- Concepts:
- Data points:
- Concrete examples:
- Possible contradictions:

### Proposed Wiki Updates

- Source page:
- Existing pages to update:
- New page candidates:
- Overview or synthesis impact:
- Index and log updates:

### Assumptions

- Emphasis:
- De-prioritized or skipped material:
- Uncertainty:
```

Ask the user whether to emphasize or skip anything unless they requested unattended ingestion. If proceeding unattended, record the assumptions in the final report.
